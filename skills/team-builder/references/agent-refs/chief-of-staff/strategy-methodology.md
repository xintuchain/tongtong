# 矩阵策略方法论

## 产品组合评估框架

### 产品健康度评分卡（每产品，月度更新）

| 维度 | 指标 | 权重 | 评分(1-5) | 说明 |
|------|------|:----:|:---------:|------|
| **流量** | 月UV增长率 | 20% | | >20%=5, 10-20%=4, 0-10%=3, 持平=2, 下降=1 |
| **留存** | D7回访率 | 20% | | >40%=5, 30-40%=4, 20-30%=3, 10-20%=2, <10%=1 |
| **变现** | 收入/利润趋势 | 15% | | 盈利=5, 接近盈亏=4, 有收入=3, 无收入=2, 亏损=1 |
| **竞争** | 差异化强度 | 15% | | 独占优势=5, 明显差异=4, 有差异=3, 同质=2, 落后=1 |
| **效率** | 投入产出比 | 15% | | 高ROI=5, 正向=4, 中等=3, 低=2, 负=1 |
| **战略** | 对矩阵协同价值 | 15% | | 核心枢纽=5, 高协同=4, 有协同=3, 独立=2, 拖累=1 |

**总分 = Σ(评分 × 权重)**

### 产品分类（简化BCG矩阵）

| 类型 | 条件 | 策略 |
|------|------|------|
| ⭐ **明星** | 高增长 + 高份额（或潜力） | 全力投入，抢市场 |
| 🐄 **现金牛** | 低增长 + 高份额 | 维护为主，收割利润，少投入 |
| ❓ **问号** | 高增长 + 低份额 | 限期验证：3个月达标→升星，否则降级 |
| 🐕 **瘦狗** | 低增长 + 低份额 | 最低维护或考虑下线，资源释放 |

## 资源分配模型

### 核心原则
CEO一个人 + AI团队，资源 = **CEO时间 + Agent算力 + 注意力**。

### 分配公式
```
产品资源占比 = (健康度得分 × 战略权重) / Σ(所有产品的 健康度×权重)
```

其中**战略权重**由CEO指定（1-3倍）：
- 3× = CEO明确标注的战略重点
- 2× = 有明确增长潜力
- 1× = 维护型

### 分配约束
| 规则 | 约束 |
|------|------|
| 单产品上限 | ≤60%（避免all-in风险） |
| 单产品下限 | ≥5%（维护最低限度） |
| 新产品试验 | 独立预留10-15%（不从现有产品扣） |
| 调整频率 | 月度review，CEO批准后生效 |

### Agent工时分配（参考）
| 角色 | 分配逻辑 |
|------|---------|
| fullstack-dev | 按产品资源占比分配coding时间 |
| content-chief | 按产品资源占比分配内容产出 |
| growth-lead | 聚焦当前AARRR最弱环节的产品 |
| data-analyst | 各产品均匀覆盖（数据不能偏科） |
| intel-analyst | 聚焦⭐明星和❓问号的竞品 |

### 资源重分配触发条件
| 信号 | 动作 |
|------|------|
| 产品健康度连续2月下降 | 减少投入，除非CEO坚持 |
| 产品健康度突破4.0 | 考虑加大投入（明星候选） |
| ❓问号产品3月未达标 | 降为🐕瘦狗，释放资源 |
| CEO新增产品 | 从预留池分配，不挤占现有 |
| 竞品重大威胁 | 临时增加相关产品投入（参谋长判断+CEO批准） |

## 跨产品协同策略

### 协同类型
| 类型 | 描述 | 示例 |
|------|------|------|
| **流量协同** | A产品用户引导到B产品 | 影视站→工具站推荐 |
| **技术协同** | 共享组件/服务 | 统一认证、共享UI库 |
| **数据协同** | 共享用户画像/行为数据 | 用户兴趣标签跨产品复用 |
| **品牌协同** | 统一品牌矩阵 | 主品牌→子品牌信任传递 |
| **SEO协同** | 站群互链+内容交叉引用 | 但要注意搜索引擎惩罚风险 |

### 协同评估
每次发现协同机会时评估：
- **收益**: 预期带来多少增量（流量/用户/收入）
- **成本**: 需要多少开发/运营投入
- **风险**: 是否有负面影响（如SEO惩罚、品牌稀释）
- **依赖**: 需要几个Agent配合，协调成本高不高

## 定价策略分析框架

### 定价三角
```
        价值感知
       /        \
      /          \
竞品价格 ——— 成本底线
```

### 定价方法选择
| 阶段 | 方法 | 说明 |
|------|------|------|
| 冷启动 | 免费/Freemium | 先要用户，不要收入 |
| 有留存 | 功能分级 | 基础免费 + 高级付费 |
| 有品牌 | 价值定价 | 按用户获得的价值定价 |
| 成熟期 | 竞品对标 | 参考竞品定价+差异化溢价 |

### 价格测试框架
- A/B测试不同价格点（需要足够流量）
- 用户调研："你愿意为X付多少"
- 竞品定价对标分析（intel-analyst 提供数据）
- Van Westendorp价格敏感度（如果能做调研的话）
