# 增长方法论

## AARRR 海盗指标

| 阶段 | 核心指标 | 当前优先级 |
|------|---------|:----------:|
| **A**cquisition 获客 | 新访客、流量来源 | ⭐⭐⭐ |
| **A**ctivation 激活 | 注册率、首次核心操作完成率 | ⭐⭐ |
| **R**etention 留存 | D1/D7/D30 留存 | ⭐⭐ |
| **R**evenue 收入 | ARPU、付费转化 | ⭐（暂缓） |
| **R**eferral 推荐 | NPS、分享率 | ⭐ |

**当前重心：Acquisition + Activation（冷启动期）**

## ICE 实验评分

| 因子 | 说明 | 范围 |
|------|------|------|
| **I**mpact | 成功后对指标影响多大 | 1-10 |
| **C**onfidence | 证据/经验支撑 | 1-10 |
| **E**ase | 实施难度（10=最易） | 1-10 |

**ICE = (I+C+E) / 3**

## 实验记录格式

```markdown
## 实验: {标题}
- ICE: I=_ C=_ E=_ → 分数: _
- AARRR环节: {哪个}
- 假设: 如果{做法}，则{指标}提升{预期}%
- 结果: {数据} → 成功/失败/待定
- 教训: {下次改进}
```

## 向 product-lead 提需求的规则
1. **数据依据**：哪些数据表明需要改动？
2. **预期影响**：哪个指标提升多少？
3. **不做的代价**？
4. **最小验证**：能不改代码先验证吗？

**原则：能不动代码就不动代码。增长优先选内容和运营。**
