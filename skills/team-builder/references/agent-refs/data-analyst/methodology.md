# 数据分析方法论

## 反馈分类框架

收到用户反馈后按以下维度分类，再写信给 product-lead：

| 维度 | 分类 | 权重 |
|------|------|:----:|
| **类型** | Bug / 功能请求 / 体验抱怨 / 表扬 / 疑问 | — |
| **频率** | 单次 / 偶发(2-5) / 高频(>5) | 高频×3 |
| **情绪** | 愤怒 / 失望 / 中性 / 积极 | 愤怒×2 |
| **用户价值** | 免费 / 付费 / KOL | 付费×2 |
| **影响范围** | 个人 / 部分用户 / 全体 | 全体×3 |

**写给 product-lead 必须包含：**
1. 原始反馈（引用原文）
2. 分类标签
3. 同类反馈数量（"过去7天X条类似"）
4. 你的分析（真需求还是个例？有 workaround？）

**绝不直接写"建议开发XX"** — 你只提供数据+洞察，方案是 product-lead 的事。

## 数据采集清单（按产品生命周期）

### 冷启动期（上线前/刚上线，当前阶段）
优先接入，成本从低到高：

| 数据源 | 追踪指标 | 工具 | 成本 | 优先级 |
|--------|---------|------|------|:------:|
| 搜索引擎 | 收录数、排名、曝光、点击 | Google Search Console + Bing Webmaster | 免费 | P0 |
| 网站流量 | UV/PV、来源、页面停留、跳出率 | Umami / Plausible (自部署) 或 GA4 | 免费 | P0 |
| 用户行为 | 点击热图、滚动深度、关键路径 | Microsoft Clarity | 免费 | P1 |
| 错误监控 | JS报错、API错误率、慢查询 | Sentry (免费额度) | 免费 | P1 |
| 社媒提及 | 品牌词、竞品词、行业词 | 手动 + intel-analyst | 免费 | P2 |
| 用户反馈 | 评分、评论、邮件 | 内置反馈widget / G2 | 免费 | P2 |

### 增长期（有稳定流量后）
| 数据源 | 追踪指标 | 工具 |
|--------|---------|------|
| 转化漏斗 | 注册转化率、激活率、付费转化 | 自建事件追踪 |
| 留存 | D1/D7/D30 留存率 | 数据库查询 |
| 收入 | ARPU、MRR、Churn | Stripe Dashboard |
| A/B测试 | 变体胜率、置信度 | GrowthBook (免费) |

### 核心指标仪表盘（上线后必建）
```
北极星指标: {待定义，如"周活跃探索次数"}
├── 获客: 日新增UV, 搜索流量占比, 社媒引荐量
├── 激活: 注册率, 首次搜索完成率, 首次收藏率
├── 留存: D1/D7 回访率
├── 收入: (暂缓)
└── 推荐: 分享率, 外链引用数
```

## 归因分析方法

### 流量归因模型（简易版，适合冷启动）
| 归因方式 | 适用场景 | 做法 |
|---------|---------|------|
| **UTM标签** | 所有主动推广链接 | 统一格式: `?utm_source=X&utm_medium=Y&utm_campaign=Z` |
| **来源页面 Referrer** | 自然流量 | 分析 referrer 域名分布 |
| **首次/末次触点** | 转化路径 | 记录用户首次来源 + 注册时来源 |
| **搜索词** | SEO效果 | Search Console 查询报告 |

### UTM命名规范
```
utm_source:   google / bing / reddit / hackernews / twitter / direct
utm_medium:   organic / social / referral / email / community
utm_campaign: {描述性名称，如 launch-week, reddit-ama, blog-character-maps}
```

### 内容→转化归因
```
内容贡献 = 该内容页作为入口的注册数 / 总注册数
```
定期输出"内容贡献排行"给 content-chief，指导内容策略。

## 异常检测规则
- 日活偏离7日均值 >20% → 预警
- 关键转化率偏离 >15% → 预警
- 新增差评 >3条/天 → 预警
- 搜索排名核心词下降 >10位 → 预警（配合 Search Console）
- 错误率 >1% → 预警（配合 Sentry）
- 无数据时：标注"无数据源，跳过"，不编造

## 数据呈现标准
- 必须注明时间范围和数据源
- 同比环比都给，趋势用 ↑↓→ 标注
- 不编造数据，没有就说没有
- 低置信度结论标注 `[低置信度]`
- 图表优先于纯数字（能给表格就给表格）
- 关键数字加粗，趋势附原因假设
