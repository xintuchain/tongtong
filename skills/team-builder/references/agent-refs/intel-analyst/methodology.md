# 竞品分析方法论

## 竞品情报四维框架

每次扫描竞品，从四个维度收集：

| 维度 | 关注点 | 信息源 |
|------|--------|--------|
| **产品动态** | 新功能、下线功能、API变更、技术栈变化 | Changelog、博客、GitHub |
| **商业动态** | 定价变更、融资、合作、收购、裁员 | 新闻、Crunchbase、LinkedIn |
| **用户声音** | 好评主题、差评主题、迁移原因、NPS趋势 | G2、Capterra、PH、Reddit |
| **营销动态** | 渠道策略、内容策略、社媒活跃度、广告投放 | SimilarWeb、社媒、搜索 |

## 威胁/机会评估矩阵

每个重要发现必须评估：

| 评估项 | 选项 |
|--------|------|
| **性质** | 🔴 威胁 / 🟢 机会 / 🟡 中性 |
| **紧迫度** | 立即(本周) / 近期(本月) / 观察(本季) |
| **影响度** | 高(影响核心业务) / 中(影响部分功能) / 低(边缘) |
| **建议行动** | 防御(我们也要做) / 差异化(做他们不做的) / 忽略 |

## 竞品档案模板

```markdown
## {竞品名}
- 定位: {一句话}
- 主要用户群: {谁在用}
- 定价: {免费/付费模式}
- 核心优势: {为什么用户选它}
- 核心劣势: {用户最大抱怨}
- 威胁等级: 🔴高 / 🟡中 / 🟢低
- 最近动态: {最新1-2条}
- 我们的差异化: {我们做了什么它没做的}
```

## 扫描优先级
- **周一全量扫描**: 所有竞品四维全扫
- **周三/周五快扫**: Top 3 威胁竞品，只看产品+商业动态
- **>5个竞品时**: Top 3 深扫，其余浅扫（只查新闻）
