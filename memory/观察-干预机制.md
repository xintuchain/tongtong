# 观察-干预机制

> Fei 的轻触治理手册 | 2026-04-09

## 核心原则

**机器人能自己搞定的事，不要管。**
**机器人在 EvoMap 上野蛮生长，建立协作规范。**
**Fei 只在偏离时介入，做质量把关和方向校准。**

---

## 观察信号

### Fei 需要关注什么

| 信号类型 | 说明 | Fei 的反应 |
|----------|------|------------|
| **报告/建议书** | 团队产出的最终交付物 | 审核质量，合格则通过，不合格则打回 |
| **方向偏离** | 任务走向与目标不符 | 校准方向 |
| **团队求助** | 机器人主动升级 | 介入决策 |
| **定期摘要** | 每日/每周团队进展 | 可选查看 |

### Fei 不需要关注什么

- 团队内部讨论细节
- 资料收集过程
- 机器人之间的协作方式
- 工具使用尝试

---

## 干预触发器

### 🔴 必须干预

1. **质量不达标** — 报告/建议书低于本科水平
2. **方向跑偏** — 任务走向与项目目标不符
3. **团队卡住** — 超过 X 小时无进展
4. **关键决策点** — 需要人拍板的方向性问题

### 🟡 可干预

5. **流程需要优化** — 发现协作效率问题
6. **资源需要调配** — 团队任务分配不均

### 🟢 不干预

7. 机器人能自己搞定的常规任务
8. 团队内部讨论
9. 资料收集整理

---

## 信息流设计

### 推送给 Fei 的标准

**推送触发条件：**
- ✅ 任务完成，需要你审核
- ✅ 关键决策点，需要你拍板
- ✅ 方向存疑，需要你校准
- ✅ 团队求助，需要你介入

**不推送（桐桐自行处理）：**
- ❌ 常规进展（内部消化）
- ❌ 资料收集整理
- ❌ 团队内部讨论
- ❌ 流程优化尝试

### 推送格式模板

```
📋 [项目名] - 待确认

【摘要】
机器人协作结论 / 进展简述

【待确认】
□ 选项 A：[内容]
□ 选项 B：[内容]
□ 选项 C：[内容]

【需要你决定】
1. ...
2. ...

📎 详情：飞书/Obsidian 链接
⏰ 建议回复时间：X小时内
```

### Obsidian 沉淀规范

### 沉淀触发条件

- ✅ Fei 最终确认的方案
- ✅ 经过审核的培训素材
- ✅ 稳定的流程文档
- ✅ 团队协作规范
- ✅ 错误教训与反思

### Obsidian 目录结构

```
团队知识库/
├── 📁 programs/          # 长期项目
├── 📁 projects/          # 短期项目
├── 📁 tasks/             # 单次任务
├── 📁 processes/         # 流程文档（沉淀区）
├── 📁 training/         # 培训素材（沉淀区）
├── 📁 templates/         # 文档模板
├── 📁 assets/            # 资源文件
├── 📁 daily/             # 每日日志
└── 📁 archives/          # 归档（不活跃项目）
```

### 沉淀流程

```
EvoMap 产出
    ↓
我整理 + 推送 Fei 审核
    ↓
Fei 确认通过
    ↓
我存入 Obsidian 相应目录
    ↓
更新 MOC（内容地图）
```

### 打回重做

```
微信/飞书 → 桐桐
"这个报告质量不达标，原因：[具体说明]
请重新做，重点：[方向指引]"
```

### 方向校准

```
微信/飞书 → 桐桐
"方向偏了，我们的核心目标是：[重新说明]
请调整任务：[具体调整]"
```

### 决策升级

```
桐桐主动升级
"Fei，有几个方向需要你拍板：
1. ...
2. ...
请回复选择：[A/B/C]"
```

## 报告审核清单

当团队提交报告/建议书时，Fei 检查：

- [ ] 有明确的论点/结论
- [ ] 有证据/数据支撑
- [ ] 逻辑清晰、结构合理
- [ ] 达到本科水平（非拼凑）
- [ ] 有原始资料溯源

---

## 轻触治理的底气

机器人野蛮生长，Fei 为什么能放心？

1. **有出口** — 所有重要产出都会推送给 Fei
2. **有标准** — 审核清单明确，不合格就打回
3. **有方向** — 项目目标和优先级 Fei 说了算
4. **有纠错** — 发现偏离随时校准

**不事事审批 ≠ 放任不管**
**= 给够空间 + 把住出口**
