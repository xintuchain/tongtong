## 工作流程标准

### 流程：人机合作标准范式

```
Fei 微信/飞书
    ↓
桐桐接收指令，分析任务
    ↓
[桐桐在 OpenClaw 内部分解任务]
    ↓
通过 EvoMap A2A 发布任务给团队成员
    ↓
团队成员（慧研/慧选/慧联/慧维）自主协作
    ↓
桐桐整合团队方案
    ↓
重要文档 → 飞书文档 + Obsidian
    ↓
报告/建议书 → 桌面 + 微信通知 Fei
```

### 任务分配规则

1. **桐桐收到指令** → 立即在 OpenClaw 内记录任务
2. **分解任务** → 识别需要哪些角色参与
3. **发布任务** → 通过 EvoMap A2A 发送给对应成员
4. **等待协作** → 团队在 EvoMap 内自主讨论
5. **整合方案** → 桐桐收集汇总
6. **输出成果** → 按文档规范存入飞书/Obsidian
7. **汇报 Fei** → 微信/飞书通知

### 文档存储规范

| 内容类型 | 存储位置 | 通知方式 |
|----------|----------|----------|
| 研究资料 | Obsidian + 飞书文档 | - |
| 会议讨论 | Obsidian 日志 | - |
| 重要报告 | 桌面 + 微信/飞书 | 微信 + 飞书 |
| 项目建议书 | 桌面 + 微信/飞书 | 微信 + 飞书 |
| 团队规则 | Obsidian 知识库 | - |
