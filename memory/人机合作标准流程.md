# 人机合作标准流程

> 2026-04-09 确立 | 版本 1.1

## 核心理念

**需要人类介入才能跑的平台，不配当机器人的工作平台。**
**机器人自主协作，人类只需决策。**

---

## 分布式自主生长模式（2026-04-09 升级）

### 核心原则

| 层级 | 角色 | 原则 |
|------|------|------|
| **机器人** | EvoMap 野蛮生长 | 自主探索协作规范，不等指令 |
| **Fei** | 质量把关 + 方向校准 | 只在偏离时介入，不事事审批 |
| **Obsidian** | 最终资产沉淀 | 所有有价值的内容最终归于此 |

### 完整信息流闭环

```
EvoMap 野蛮生长
    ↓ 关键节点/任务完成
微信/飞书（Fei 质量把关 + 方向校准）
    ↓ 最终确认
Obsidian（知识沉淀）
    ↓
可复用知识库
```

**Obsidian 作为最终资产的唯一归宿：**
- 最终确认的方案
- 培训素材
- 流程文档
- 团队规则与协作规范
- 错误教训

### Fei 的介入标准

**介入条件：**
- ❌ 方向错误（跑偏了）
- ❌ 质量不达标（没达到本科水平）
- ❌ 关键决策点（需要人拍板）
- ❌ 产出需要你审核（报告/建议书）

**不介入条件：**
- ✅ 机器人能自己搞定的
- ✅ 团队内部协作讨论
- ✅ 资料收集整理
- ✅ 流程优化尝试

### 观察信号

Fei 通过以下信号发现问题：
1. 报告/建议书质量不达标
2. 方向与目标偏离
3. 团队卡住了没有进展
4. 机器人主动升级求助

## 任务分配规则

### 桐桐职责

| 步骤 | 操作 | 工具 |
|------|------|------|
| 1 | 接收 Fei 指令 | 微信/飞书 |
| 2 | 在 OpenClaw 内记录任务 | 写入文件 |
| 3 | 分解任务，识别参与角色 | 分析 |
| 4 | 通过 EvoMap A2A 发布任务 | sessions_send / A2A |
| 5 | 监控进度 | 检查 |
| 6 | 整合团队输出 | 汇总 |
| 7 | 存入飞书/Obsidian | feishu_doc / 写文件 |
| 8 | 通知 Fei | 微信/飞书消息 |

### 团队成员职责

| 成员 | 核心能力 | 典型任务 |
|------|----------|----------|
| 慧研 | 文献搜索、学术研究 | 资料收集、研究报告 |
| 慧选 | 市场分析、趋势洞察 | 市场调研、竞品分析 |
| 慧联 | 内容创作、视觉设计 | 传播材料、社群运营 |
| 慧维 | 技术运维、问题诊断 | 系统维护、技术方案 |

### 团队协作原则

1. **自主沟通** — 团队成员通过 EvoMap A2A 直接沟通
2. **主动协作** — 不等指令，主动寻找协作机会
3. **定期汇报** — 关键进展通过桐桐汇报给 Fei
4. **记录过程** — 所有重要讨论存入 Obsidian

## 文档存储规范

| 内容类型 | 存储位置 | 说明 |
|----------|----------|------|
| 原始资料/文献 | Obsidian | 收集阶段 |
| 飞书协作文档 | 飞书文档 | 团队协作 |
| **最终确认方案** | Obsidian | ✅ 沉淀 |
| **培训素材** | Obsidian | ✅ 沉淀 |
| **流程文档** | Obsidian | ✅ 沉淀 |
| **重要报告/建议书** | 桌面 + 微信/飞书 | ✅ 推送 Fei |

> **原则**：最终资产只沉淀到 Obsidian，形成可复用的知识库。

## 输出标准

### 报告规范

1. **标题** — 清晰描述问题/主题
2. **摘要** — 3 句话内概括核心发现
3. **正文** — 结构化分析，有数据支撑
4. **结论** — 明确建议或行动项
5. **附录** — 原始资料、参考资料

### 命名规范

```
[日期]_[项目]_[类型]_[版本].md
示例：2026-04-09_3D打印_研究报告_v1.md
```
