### 时间框架

| 报告级别 | 人类时间 | AI 时间（可接受） |

### 核心原则

4. **独立见解与质疑** — 形成自己的见解，质疑不合理的结论
5. **引用规范** — 区分原创与引用，标注出处

## 错误图书馆（2026-04-07）

### ❌ 频道 ID 配置错误（严重，影响所有 Cron Job）

**症状**：所有子代理 cron 任务持续 401 Unauthorized，无法发送/读取 Discord 消息。

**根本原因**：cron job 的 delivery.to 使用了旧的"常规"频道 ID `1487386985852178617`，但该频道不属于任何 Bot。

**正确配置**：
| Bot | 频道 ID | 名称 |
|-----|---------|------|
| 桐桐 | `1489414235032125581` | 桐桐 |
| 慧研 | `1489414235661533336` | 慧研 |
| 慧选 | `1489414235279589508` | 慧选 |
| 慧联 | `1489414234939854859` | 慧联 |
| 慧维 | `1489414235481051338` | 慧维 |

**修复方法**：
1. 用 `cron list` 查看所有 jobs
2. 用 `cron update --jobId XXX --patch '{"delivery":{"to":"channel:正确ID"}}'` 逐个修复
3. 如果 payload 消息里有硬编码 curl 命令，也要更新消息内容

**教训**：建立任何 channel 后，立即记录 channel ID 到 TOOLS.md，而不是写在任务里。
