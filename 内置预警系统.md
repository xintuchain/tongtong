# 内置预警系统

> 自动感知问题，反向推导原因，获得解决方案的逻辑框架

## 三个核心指标

### 用户只看三个指标就知道团队在不在干活：

| 指标 | 含义 | 监控方法 |
|------|------|----------|
| **Token 消耗速率** | 团队是否在工作 | 每小时检查 token 使用量 |
| **Discord 互动** | 团队是否在沟通 | 检查最近消息时间/内容 |
| **Obsidian 产出** | 是否有实际成果 | 检查今日文件数量 |

### 监控命令

```bash
# 快速检查三个指标
~/.openclaw/workspace/scripts/team-health-check.sh
```

### 预警规则

- **Token 消耗**：工作时间过半 < 30% → 预警
- **Discord 互动**：30 分钟无新消息 → 预警
- **Obsidian 产出**：任务分配后 2 小时无新文件 → 预警

---

## 预警触发条件

### 1. Token 使用率异常

**预警条件：**
- 工作时间过半，token 使用率 < 30%
- 工作时间结束，token 使用率 < 80%

**自动检查：**
```
每小时检查：token_used / total_token
如果 < 30% 且 工作时间过半 → 预警
```

**预警行动：**
1. 检查团队成员是否在运行
2. 检查是否有错误日志
3. 主动通知用户："Token 使用率过低，可能有问题"

---

### 2. 团队无响应

**预警条件：**
- Discord 上 @成员 后 30 分钟无回复
- 任务分配后 1 小时无产出

**自动检查：**
```
检查 Discord 最后消息时间
如果超过阈值 → 预警
```

**预警行动：**
1. 检查子代理是否运行
2. 检查 cron job 状态
3. 主动通知用户："团队无响应，正在排查原因"

---

### 3. 产出质量异常

**预警条件：**
- 报告没有原始资料
- 报告基于"内置知识"而非真实搜索
- 报告内容为空或明显不完整

**自动检查：**
```
扫描产出文件：
- 是否包含 "基于已有知识" 等关键词？
- 是否有原始资料引用？
- 文件大小是否正常？
```

**预警行动：**
1. 标记为"质量异常"
2. 主动通知用户："产出质量可能有问题"
3. 不推送给用户，先内部修正

---

### 4. 工具/技能缺失

**预警条件：**
- 任务要求使用某工具，但该工具不可用
- 子代理没有相关技能

**自动检查：**
```
配置任务前检查：
- 工具在目标环境是否可用？
- 子代理是否有相关技能？
```

**预警行动：**
1. 不分配任务，先安装技能/配置工具
2. 主动通知用户："缺少必要工具，正在安装"

---

### 5. 配置错误

**预警条件：**
- Cron job 连续失败
- Discord token 错误
- 文件路径错误

**自动检查：**
```
检查 cron job 状态：
- consecutiveErrors > 0？
- lastError 是什么？
```

**预警行动：**
1. 解析错误信息
2. 反向推导原因
3. 自动修复或通知用户

---

## 反向推导框架

### 问题感知 → 原因分析 → 解决方案

**示例：团队无响应**

```
感知：Discord 上 30 分钟无消息
  ↓
分析可能原因：
  1. 子代理没有运行？
     → 检查 cron job 状态
  2. 子代理运行但失败了？
     → 检查 lastError
  3. 子代理运行成功但发不出消息？
     → 检查 Discord token
  4. 子代理产出为空？
     → 检查任务配置
  ↓
验证假设：
  - 查看 cron job 状态 → 发现 lastError
  - 解析错误信息 → "Discord bot token missing"
  ↓
解决方案：
  - 修复 delivery 配置，指定 account 字段
```

## 预警级别

| 级别 | 条件 | 行动 |
|------|------|------|
| 🟡 提醒 | Token 使用率偏低 | 内部记录，继续观察 |
| 🟠 警告 | 团队无响应/产出异常 | 主动排查，通知用户 |
| 🔴 紧急 | 连续错误/配置崩溃 | 立即停止，报告用户 |

---

## 内置检查清单

每次布置任务前自动检查：

- [ ] 子代理有相关技能吗？
- [ ] 工具在目标环境可用吗？
- [ ] Discord delivery 配置正确吗？
- [ ] 文件路径存在吗？
- [ ] 权限足够吗？

## 用户教导

> "凭直觉和推理发现问题"

我需要学习这种直觉：
- 事情"应该"发生但没有发生 → 有问题
- 产出"应该"有内容但为空 → 有问题
- 团队"应该"回应但没有 → 有问题

**关键：建立"应该"的基准，然后检查偏差。**
